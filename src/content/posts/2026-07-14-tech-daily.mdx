Node.js 让前端开发者也能写后端，但这既是优势也是陷阱——很多人用 Node.js 写出了「前端的后端」：回调满天飞、错误处理随缘、类型全靠猜、大项目难以维护。

这篇文章带你从 0 到 1 搭建一个**生产级 Node.js 后端**。

技术栈：

- **运行时**：Node.js 18+（支持原生 fetch）
- **语言**：TypeScript（严谨的类型系统）
- **框架**：Express（最流行的 Node.js 框架）
- **数据库 ORM**：Prisma（类型安全、开发体验好）
- **认证**：JWT（JSON Web Token）
- **测试**：Vitest（轻量快速）

## 第四章：路由、控制器、服务三层架构

后端代码的关键在于**分层**——每层只做一件事：

```
路由（routes）    → 注册 URL 和方法，调用控制器
控制器（controller） → 解析请求参数，调用服务层，处理响应
服务层（service）  → 业务逻辑，操作数据库，纯函数式
```

### 路由层

```typescript
// src/modules/users/user.routes.ts
import { Router } from 'express';
import * as userController from './user.controller.js';
import { authenticate } from '../../middleware/auth.js';
import { validate } from '../../middleware/validate.js';
import { updateUserSchema } from './user.validation.js';

export const userRoutes = Router();

userRoutes.get('/me', authenticate, userController.getProfile);
userRoutes.patch('/me', authenticate, validate(updateUserSchema), userController.updateProfile);
userRoutes.get('/', authenticate, userController.listUsers);
```

### 控制器层

```typescript
// src/modules/users/user.controller.ts
import { Request, Response, NextFunction } from 'express';
import * as userService from './user.service.js';

export async function getProfile(req: Request, res: Response, next: NextFunction) {
  try {
    const userId = req.user!.id;
    const user = await userService.getUserById(userId);
    res.json({ data: user });
  } catch (error) {
    next(error);
  }
}

export async function updateProfile(req: Request, res: Response, next: NextFunction) {
  try {
    const userId = req.user!.id;
    const user = await userService.updateUser(userId, req.body);
    res.json({ data: user });
  } catch (error) {
    next(error);
  }
}

export async function listUsers(req: Request, res: Response, next: NextFunction) {
  try {
    const users = await userService.listUsers();
    res.json({ data: users });
  } catch (error) {
    next(error);
  }
}
```

### 服务层

```typescript
// src/modules/users/user.service.ts
import { prisma } from '../../lib/prisma.js';

export async function getUserById(id: number) {
  const user = await prisma.user.findUnique({
    where: { id },
    select: { id: true, name: true, email: true, createdAt: true },
  });
  if (!user) throw new NotFoundError('用户不存在');
  return user;
}

export async function updateUser(id: number, data: { name?: string }) {
  return prisma.user.update({
    where: { id },
    data,
    select: { id: true, name: true, email: true, createdAt: true },
  });
}

export async function listUsers() {
  return prisma.user.findMany({
    select: { id: true, name: true, email: true, createdAt: true },
    orderBy: { createdAt: 'desc' },
    take: 20,
  });
}
```

## 第六章：JWT 认证

```typescript
// src/lib/jwt.ts
import jwt from 'jsonwebtoken';
import { env } from '../config/env.js';

interface TokenPayload {
  userId: number;
  email: string;
}

export function signToken(payload: TokenPayload): string {
  return jwt.sign(payload, env.JWT_SECRET, {
    expiresIn: env.JWT_EXPIRES_IN,
  });
}

export function verifyToken(token: string): TokenPayload {
  return jwt.verify(token, env.JWT_SECRET) as TokenPayload;
}
```

```typescript
// src/middleware/auth.ts
import { Request, Response, NextFunction } from 'express';
import { verifyToken } from '../lib/jwt.js';

// 扩展 Express 的 Request 类型
declare global {
  namespace Express {
    interface Request {
      user?: { id: number; email: string };
    }
  }
}

export function authenticate(req: Request, res: Response, next: NextFunction) {
  const header = req.headers.authorization;
  if (!header?.startsWith('Bearer ')) {
    return res.status(401).json({ error: '未提供认证令牌' });
  }

  try {
    const token = header.slice(7);
    const payload = verifyToken(token);
    req.user = { id: payload.userId, email: payload.email };
    next();
  } catch {
    return res.status(401).json({ error: '令牌无效或已过期' });
  }
}
```

```typescript
// src/modules/auth/auth.service.ts
import bcrypt from 'bcrypt';
import { prisma } from '../../lib/prisma.js';
import { signToken } from '../../lib/jwt.js';

export async function register(email: string, password: string, name: string) {
  const existing = await prisma.user.findUnique({ where: { email } });
  if (existing) throw new ConflictError('邮箱已被注册');

  const hashed = await bcrypt.hash(password, 10);
  const user = await prisma.user.create({
    data: { email, password: hashed, name },
  });

  const token = signToken({ userId: user.id, email: user.email });
  return { token, user: { id: user.id, email: user.email, name: user.name } };
}

export async function login(email: string, password: string) {
  const user = await prisma.user.findUnique({ where: { email } });
  if (!user) throw new UnauthorizedError('邮箱或密码错误');

  const valid = await bcrypt.compare(password, user.password);
  if (!valid) throw new UnauthorizedError('邮箱或密码错误');

  const token = signToken({ userId: user.id, email: user.email });
  return { token, user: { id: user.id, email: user.email, name: user.name } };
}
```

## 第七章：全局错误处理

```typescript
// src/middleware/error.ts
import { Request, Response, NextFunction } from 'express';

export class AppError extends Error {
  constructor(
    public statusCode: number,
    message: string,
  ) {
    super(message);
    this.name = this.constructor.name;
  }
}

export class NotFoundError extends AppError {
  constructor(message = '资源不存在') {
    super(404, message);
  }
}

export class UnauthorizedError extends AppError {
  constructor(message = '未授权') {
    super(401, message);
  }
}

export class ConflictError extends AppError {
  constructor(message = '资源冲突') {
    super(409, message);
  }
}

export class ValidationError extends AppError {
  constructor(message = '请求参数错误') {
    super(400, message);
  }
}

export function errorHandler(err: Error, _req: Request, res: Response, _next: NextFunction) {
  if (err instanceof AppError) {
    return res.status(err.statusCode).json({ error: err.message });
  }

  // 未预料的错误
  console.error('未处理的错误:', err);
  return res.status(500).json({ error: '服务器内部错误' });
}
```

## 第八章：请求验证

```typescript
// src/middleware/validate.ts
import { Request, Response, NextFunction } from 'express';
import { ValidationError } from './error.js';

type ValidationSchema = {
  [key: string]: (value: unknown) => string | null;
};

export function validate(schema: ValidationSchema) {
  return (req: Request, _res: Response, next: NextFunction) => {
    const errors: string[] = [];

    for (const [field, validator] of Object.entries(schema)) {
      const error = validator(req.body[field]);
      if (error) errors.push(`${field}: ${error}`);
    }

    if (errors.length > 0) {
      return next(new ValidationError(errors.join('; ')));
    }

    next();
  };
}

// src/modules/auth/auth.validation.ts
export const registerSchema = {
  email: (v: unknown) => (!v ? '邮箱必填' : typeof v !== 'string' ? '邮箱格式错误' : null),
  password: (v: unknown) =>
    !v ? '密码必填' : typeof v !== 'string' || v.length < 6 ? '密码至少 6 位' : null,
  name: (v: unknown) => (!v ? '名称必填' : null),
};
```

## 避坑清单

| 坑 | 正确做法 |
|----|---------|
| 不设 strict: true | TypeScript 的 strict 模式必须开，否则 `any` 偷偷溜进来 |
| 用 `any` 糊弄类型 | 数据库查询结果、请求体都要定义接口或用 Prisma 自动生成类型 |
| 错误处理写在每个路由里 | 全局 `errorHandler` 中间件统一处理，路由里只 `next(error)` |
| 密码明文存储 | 用 bcrypt 加盐哈希，永远不要自己写哈希算法 |
| JWT 不过期 | 设短有效期（7 天），配合刷新令牌 |
| 不验证请求参数 | 每个 POST/PATCH 接口都要验证输入（类型、长度、格式） |
| 生产环境跑 `tsx watch` | 用 `tsc` 编译后用 `node dist/index.js` 运行 |
| 不用环境变量 | 敏感信息（数据库密码、JWT Secret）走 `.env`，别硬编码 |
| 忽略 Prisma 的 N+1 问题 | 用 `include` 或 `Prisma Promise` 批量加载关联数据 |
| 不加速率限制 | 生产环境加 `express-rate-limit` 防暴力破解 |
| 不写测试 | 核心逻辑（认证、业务服务）用单元测试覆盖，API 用集成测试 |

## 小结

你刚刚建了一个**分层清晰、类型安全、有错误处理、有认证、有测试**的 Node.js 后端。这不是一个「Hello World」项目，而是一个生产级模板。

下一步你可以加什么：

- **速率限制**：`express-rate-limit` 防止恶意请求
- **日志**：`pino` 结构化日志，接入日志平台
- **任务队列**：`BullMQ` 处理异步任务（发邮件、生成 PDF）
- **文件上传**：`multer` + 云存储（S3/CDN）
- **API 文档**：`Swagger/OpenAPI` 自动生成接口文档
- **Docker 化**：写 `Dockerfile` + `docker-compose.yml` 一键部署

Node.js 后端的核心不是框架 API——是分层思维、错误处理、类型安全和测试习惯。框架会过时，但这些习惯不会。
