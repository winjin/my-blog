GoFrame（简称 gf）是一款模块化、高性能、生产级的 Go 基础开发框架。你可以把它理解为 **Go 语言的 Spring Boot**——大而全，开箱即用。

与 Gin、Echo 等轻量级框架不同，GoFrame 不只是一个路由器，它提供了完整的开发生态：ORM、缓存、日志、配置、校验、定时任务、消息队列、单元测试、Swagger 文档生成……几乎所有 Web 项目需要的基础设施它都内置了。

## 第一章：安装与脚手架

### 安装 gf CLI

```bash
# 方式一：推荐，一键安装
wget -O gf "https://github.com/gogf/gf/releases/latest/download/gf_$(go env GOOS)_$(go env GOARCH)" && chmod +x gf && ./gf install -y && rm ./gf

# 方式二：Go install
go install github.com/gogf/gf/cmd/gf/v2@latest

# 验证
gf -v
# v2.10.x
# Welcome to GoFrame!
```

> **zsh 用户注意**：`gf` 和 `git fetch` 的别名冲突。安装后在 `.zshrc` 中添加 `alias gf="gf"` 然后重启终端或执行 `source ~/.zshrc`。

### 创建项目

```bash
# 初始化项目脚手架
gf init my-blog -u   # -u 更新框架到最新版本
cd my-blog
```

## 第二章：目录结构与核心概念

```text
my-blog/
├── main.go                  # 入口文件
├── go.mod / go.sum          # Go 模块文件
├── api/                     # 对外接口（DTO：输入/输出数据结构）
│   └── blog/
│       └── v1/
│           └── post.go
├── internal/                # 内部逻辑（不对外暴露）
│   ├── cmd/
│   │   └── cmd.go           # 启动命令 + 路由注册
│   ├── controller/          # 控制器（处理请求、解析参数、调用 Logic）
│   ├── logic/               # 业务逻辑层
│   ├── model/               # 内部数据结构（DO/Entity）
│   ├── dao/                 # 数据访问层（自动生成）
│   └── service/             # 接口定义层
├── manifest/
│   ├── config/config.yaml   # 配置文件
│   ├── docker/Dockerfile    # Docker 部署文件
│   └── deploy/              # Kubernetes 部署文件
├── resource/
│   └── template/            # 模板文件
└── hack/                    # 工具脚本
```

**核心设计理念：**

```text
api/      定义「别人怎么调用我」（请求/响应结构）
controller 解「别人请求了我，我转发给谁」
logic/    定义「真正的业务逻辑怎么做」
dao/      定义「数据怎么存取」
model/    定义「内部数据长什么样」
service/  定义「logic 对外承诺的接口」
```

## 第三章：第一个 API

### 定义数据结构

```go
// api/blog/v1/post.go
package v1

import "github.com/gogf/gf/v2/frame/g"

// 创建文章请求
type CreatePostReq struct {
    g.Meta   `path:"/posts" method:"post" tags:"Post" summary:"创建文章"`
    Title    string `v:"required|length:1,200" dc:"文章标题"`
    Content  string `v:"required" dc:"文章内容"`
    Category string `v:"required|in:tech,life,economy" dc:"分类"`
}

// 创建文章响应
type CreatePostRes struct {
    Id uint64 `json:"id"`
}
```

### 实现控制器

```go
// internal/controller/blog/post.go
package blog

import (
    "context"
    "my-blog/api/blog/v1"
    "my-blog/internal/logic"
)

type PostController struct{}

func NewPostController() *PostController {
    return &PostController{}
}

func (c *PostController) Create(ctx context.Context, req *v1.CreatePostReq) (res *v1.CreatePostRes, err error) {
    id, err := logic.Post().Create(ctx, req)
    if err != nil {
        return nil, err
    }
    return &v1.CreatePostRes{Id: id}, nil
}
```

### 业务逻辑层

```go
// internal/logic/post.go
package logic

import (
    "context"
    "my-blog/api/blog/v1"
    "my-blog/internal/dao"
    "my-blog/internal/model/entity"
)

type postLogic struct{}

func Post() *postLogic { return &postLogic{} }

func (l *postLogic) Create(ctx context.Context, req *v1.CreatePostReq) (id uint64, err error) {
    result, err := dao.Post.Ctx(ctx).Insert(entity.Post{
        Title:    req.Title,
        Content:  req.Content,
        Category: req.Category,
    })
    if err != nil {
        return 0, err
    }
    lastId, _ := result.LastInsertId()
    return uint64(lastId), nil
}
```

### 注册路由

```go
// internal/cmd/cmd.go
package cmd

import (
    "context"
    "github.com/gogf/gf/v2/frame/g"
    "github.com/gogf/gf/v2/net/ghttp"
    "github.com/gogf/gf/v2/os/gcmd"
    "my-blog/internal/controller/blog"
)

var Main = &gcmd.Command{
    Name:  "main",
    Brief: "启动 HTTP 服务",
    Func: func(ctx context.Context, parser *gcmd.Parser) error {
        s := g.Server()

        // 使用标准返回格式中间件
        s.Use(ghttp.MiddlewareHandlerResponse)

        // 分组注册
        s.Group("/api", func(group *ghttp.RouterGroup) {
            group.Bind(
                blog.NewPostController(),
            )
        })

        s.Run()
        return nil
    },
}
```

### 启动

```bash
go run main.go
# 访问 http://localhost:8000/api/posts
# Swagger: http://localhost:8000/swagger
```

### 常用操作

```go
import (
    "github.com/gogf/gf/v2/frame/g"
    "github.com/gogf/gf/v2/database/gdb"
)

// 查询
func GetPost(ctx context.Context, id uint64) (*entity.Post, error) {
    var post *entity.Post
    err := dao.Post.Ctx(ctx).Where("id", id).Scan(&post)
    return post, err
}

// 列表查询
func ListPosts(ctx context.Context, category string, page, size int) ([]*entity.Post, int, error) {
    m := dao.Post.Ctx(ctx).OrderDesc("created_at")
    if category != "" {
        m = m.Where("category", category)
    }
    total, err := m.Count()
    if err != nil {
        return nil, 0, err
    }
    var posts []*entity.Post
    err = m.Page(page, size).Scan(&posts)
    return posts, total, err
}

// 事务
func CreateWithTags(ctx context.Context, post *entity.Post, tags []string) error {
    return dao.Post.Transaction(ctx, func(ctx context.Context, tx *gdb.TX) error {
        id, err := tx.Insert("post", g.Map{
            "title":    post.Title,
            "content":  post.Content,
            "category": post.Category,
        })
        if err != nil {
            return err
        }
        lastId, _ := id.LastInsertId()
        for _, tag := range tags {
            _, err = tx.Insert("post_tag", g.Map{
                "post_id": lastId,
                "tag":     tag,
            })
            if err != nil {
                return err
            }
        }
        return nil
    })
}
```

## 第五章：数据校验

GoFrame 内置了强大的验证组件 `gvalid`，直接在结构体标签中定义规则：

```go
type RegisterReq struct {
    g.Meta   `path:"/auth/register" method:"post"`
    Email    string `v:"required|email" dc:"邮箱"`
    Password string `v:"required|password:6,18" dc:"密码（6-18位）"`
    Age      int    `v:"required|between:1,150" dc:"年龄"`
    Phone    string `v:"phone" dc:"手机号"`
}

// 自定义错误消息
type LoginReq struct {
    Email    string `v:"required|email#邮箱必填|邮箱格式错误"`
    Password string `v:"required|length:6,18#密码必填|密码长度6-18位"`
}
```

**常用验证规则：**

| 规则 | 说明 |
|------|------|
| `required` | 必填 |
| `length:6,18` | 长度范围 |
| `min:1` / `max:100` | 数字范围 |
| `in:tech,life` | 枚举值 |
| `email` | 邮箱格式 |
| `phone` | 手机号 |
| `password:6,18` | 密码强度 |
| `same:field` | 与另一字段相同（确认密码） |
| `regex:pattern` | 正则表达式 |

## 第六章：中间件

```go
// internal/middleware/cors.go
package middleware

import "github.com/gogf/gf/v2/net/ghttp"

func CORS(r *ghttp.Request) {
    r.Response.CORSDefault()
    r.Middleware.Next()
}

// internal/middleware/auth.go
func Auth(r *ghttp.Request) {
    token := r.Header.Get("Authorization")
    if token == "" {
        r.Response.WriteJson(g.Map{
            "code":    401,
            "message": "未授权",
        })
        r.Exit()
    }
    // 验证 token...
    r.Middleware.Next()
}
```

```go
// 在 cmd.go 中注册
s.Group("/api", func(group *ghttp.RouterGroup) {
    group.Middleware(middleware.CORS)
    group.Middleware(middleware.Auth)
    group.Bind(blog.NewPostController())
})
```

### Dao 层自动生成

```bash
# 从数据库表自动生成 dao/model/controller 代码
gf gen dao
gf gen controller
```

这是 GoFrame 的一大特色——**代码生成工具链**。连接数据库后，自动生成：

```text
- dao/    → 增删改查基础方法（自动生成）
- model/  → entity 和 DO 结构体（自动生成）
- api/    → 接口 DTO 可半自动生成
```

## 避坑清单

| 坑 | 正确做法 |
|----|---------|
| 把 GoFrame 当 Gin 用 | GoFrame 是约定优于配置的**全栈框架**，按它的分层（api/controller/logic/dao/model）来组织代码 |
| 忽略 `gf gen dao` | 数据库表变了就重新 `gf gen dao`，dao 层代码自动生成，不要手写 |
| 不设数据校验 `v:` 标签 | GoFrame 的 `v:` 标签可以在 API 入口统一校验，不要在 controller 里手写 if/else 校验 |
| 不理解 `g.Meta` 的作用 | `g.Meta` 是路由注册、OpenAPI 文档生成的桥梁，每个 API 结构体都必须有 |
| 直接用 `g.DB()` 裸写 SQL | 用 `dao` 层的方法，类型安全、自动防注入、支持自动 Scan 到结构体 |
| 在 controller 里写业务逻辑 | controller 只做「取参数→调 logic→返结果」，业务逻辑放 logic 层 |
| 忽略 Swagger 文档 | 只要 API 结构体定义完整，`/swagger` 页面自动生成，无需额外配置 |
| 手动管理数据库迁移 | 用 `gf gen dao` 自动化，或配合 `gdb` 的自动建表功能 |
| gf 和 zsh 别名冲突 | 在 `.zshrc` 加 `alias gf="gf"` |
| 所有环境用同一套配置 | 利用多环境配置自动覆盖机制（dev/prod 自动读取不同配置） |

## 小结

GoFrame 和 Gin 的最大区别不在于「谁快」，而在于**项目组织方式的哲学**。

- **Gin**：给你一个路由器，其他你自己选（ORM 用 GORM、校验用 go-playground/validator、配置用 viper……）
- **GoFrame**：给你一套完整的企业级项目模板 + 开发工具链 + 全栈组件

```text
适合 GoFrame 的场景：
- 企业级 Web 应用
- 微服务（GoFrame 内置了微服务支持）
- 团队协作（约定统一，新人上手快）
- 需要快速交付的项目（代码生成 + 内置组件）

适合 Gin 的场景：
- 极轻量的 API 服务
- 已选型好第三方组件
- 对框架侵入性敏感
```

用 GoFrame 写 Go 后端，感觉像是从「拼乐高」变成了「用宜家家具」——你不用自己选每个零件，但它给了你一套经过验证的、完整可用的方案。
