## 第一章：代码补全能力

### GitHub Copilot

最早的 AI 补全工具，深度集成 GitHub 生态：

```typescript
// 写下注释，Copilot 自动补全实现
// 将数组中的对象按 createdAt 字段降序排列
const sorted = items.sort((a, b) =>
  new Date(b.createdAt).getTime() - new Date(a.createdAt).getTime()
);
```

**优势**：VS Code 原生体验、GitHub 上下文（PR/Issue）  
**局限**：补全质量取决于上下文窗口，复杂场景不如 Cursor

### Cursor Tab

多行预测 + 编辑意图感知：

```typescript
// 你在一处改了变量名 userName → displayName
// Cursor Tab 自动预测文件中所有相关位置的修改
```

**优势**：多行补全、编辑预测、跨文件上下文  
**局限**：需要使用 Cursor IDE（不能在 VS Code 中用）

### Claude Code & Codex

CLI 工具不做实时补全，而是整块生成：

```bash

# → 直接生成完整实现文件
```

**定位不同**：不是逐行辅助，而是整个任务委托

## 第三章：多文件编辑

这是各工具差异最大的地方。

### Copilot Edits

2025 年新增的多文件编辑功能：
- 在编辑器内选择要修改的文件
- 描述修改意图
- 逐文件展示 diff

评价：**能用，但不够自主**——你需要手动指定哪些文件需要改。

### Cursor Composer

Cursor 的核心竞争力：
- 描述需求后自动识别相关文件
- 同时展示多文件 diff
- Agent 模式可执行命令验证

评价：**目前 IDE 形态中最强的多文件编辑体验**。

### Claude Code

终端代理的自主操作：
- 自动搜索需要修改的文件
- 读取→分析→修改→验证的完整循环
- 可以运行测试确认修改正确

评价：**最自主**，适合「描述目标，剩下交给 AI」的场景。

### Codex CLI

沙箱内的全自动操作：
- `full-auto` 模式下完全自主
- 网络隔离保证安全
- 但可撤销性不如 Claude Code 的 Worktree

评价：**安全优先的自动化**，适合信任模型但需要安全保障的场景。

## 第八章：实际工作流对比

### 场景：实现一个用户头像上传功能

**Copilot 方式**：
```
1. 手动创建文件
2. 写注释描述每个函数
3. Copilot 逐个补全实现
4. 手动在各文件间跳转补全
时间：~30 分钟
```

**Cursor 方式**：
```
1. Composer: "实现用户头像上传，前端裁剪 + 后端存 S3 + 数据库字段"
2. 审查多文件 diff
3. 一键接受 → Agent 运行测试
时间：~10 分钟
```

**Claude Code 方式**：
```
1. claude "实现用户头像上传功能..."
2. Claude 自动分析项目结构
3. 创建组件/路由/migration
4. 自动运行测试验证
5. 展示修改摘要等你确认
时间：~8 分钟
```

**Codex 方式**：
```
1. codex --approval-mode auto-edit "实现用户头像上传..."
2. Codex 在沙箱内实现
3. 展示改动等你确认
时间：~8 分钟
```

## 总结

没有「最好」的工具，只有最适合当前任务的工具：

- **Copilot** — 最低门槛，VS Code 用户的默认选择
- **Cursor** — 最佳编辑体验，IDE 党的终极选择
- **Claude Code** — 最强自主能力，重度 AI 用户的主力
- **Codex** — 最安全开源，合规需求的首选

它们不是互斥的——最聪明的做法是根据场景混合使用。就像你不会只用一把螺丝刀修所有东西一样。
